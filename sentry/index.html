Effortless monitoring

Create your own monitoring service with just a few lines of code. Get instantly notified about any possible system failures or downtimes and collect the custom data to build your own metrics.

view_module

Convention over configuration

Make use of the provided fluent API to quickly define a set of so called watchers responsible for monitoring the resource. You may choose between Web/API, MSSQL and MongoDB watchers and there's many more coming in the near future.

code

Easily extendable

Need to provide monitoring service for some custom resource that should work as a part of the Sentry? It couldn't have been any simpler, just implement a single IWatcher interface and add your watcher to the Sentry.
